## [Google密钥设计缺陷或致Gemini API密钥泄露](https://trufflesecurity.com/blog/google-api-keys-werent-secrets-but-then-gemini-changed-the-rules) `#4`
> **Truffle Security** 披露 **Google Cloud** 存在隐患：开发者在项目中启用 `Gemini` 后，原本用于 `Maps` 等服务且被视为可公开的旧版 API 密钥，会自动获得访问 `Gemini` 的权限。**Google** 已针对新密钥实施默认权限限制，并阻断已泄露密钥。

据第三方安全研究机构 **Truffle Security** 的报告，**Google Cloud** 存在一项安全风险。该风险源于 **Google** 对 API key 的处理方式：长期以来，用于 `Maps` 等服务的 API key 被视为可公开的标识符，但随着 `Gemini API` 的推出，**Google** 沿用单一密钥格式。当开发者为现有项目启用 `Gemini API` 时，项目中所有旧密钥会自动获得访问私有数据、调用模型及产生费用的权限。

针对该问题，**Google** 官方已采取措施。通过 **Google AI Studio** 创建的新密钥将默认限制为仅访问 `Gemini API`，防止跨服务滥用；系统将默认阻断被发现已泄露且用于 `Gemini API` 的密钥；并计划在识别到密钥泄露时主动通知用户。
